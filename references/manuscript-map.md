# Manuscript Map

This skill is derived from the manuscript _"独一无二"的活动：法律行业市场品牌实战手册_. Use this file first to decide which reference to read next instead of opening everything.

## Core Thesis

- A good legal-industry event is not just a gathering. It is a system for turning professional capability into a felt experience.
- The real output is not only attendance or applause. It is trust, memory, relationship depth, and reusable content assets.
- The biggest failure mode is organizer-centered design: broad topics, fixed templates, weak interaction, and no post-event carryover.

## Chapter Map

### Chapter 1: Observation

Use when diagnosing why an event feels generic or underperforms.

- Typical problems: topic homogeneity, rigid agenda templates, outdated formats, weak dissemination, low attention.
- Key shift: treat events as a brand-and-trust amplifier rather than a standalone marketing task.

### Chapter 2: Planning

Use when shaping a new event from a vague idea.

- Clarify goal, audience, positioning, naming, content, guests, format, and structural arc.
- Three anchor questions:
  - Who must remember the event?
  - What should they remember?
  - How will success be judged?

### Chapter 3: Proposal

Use when you need a plan that can align a team or persuade stakeholders.

- Proposal is a shared map internally and a persuasion tool externally.
- The manuscript emphasizes eight core modules:
  - Cover and one-line positioning
  - Background and goals
  - Positioning and audience
  - Framework and highlights
  - Agenda design
  - Guest and resource plan
  - Propagation and sponsorship
  - Execution and safeguards

### Chapter 4: Execution

Use when the event concept is already set and the real challenge is delivery.

- Execution is judged through details, timing, and ownership.
- Focus on milestone control, site flow, signage, staff coordination, and post-event asset capture.

### Chapter 5: Design

Use when briefing designers or planning stage, signage, and on-site atmosphere.

- Visuals are part of the first experience.
- Design quality is not only aesthetics. It sets tone, signals professionalism, and helps the event become memorable.

### Chapter 6: Experience

Use when you need to increase emotional retention, networking quality, or participant comfort.

- Think in three stages: prelude, live experience, afterglow.
- Build around differentiated participant needs and at least one strong memory point.

### Chapter 7: Propagation (传播)

Use when the user asks for warm-up, live content, recap, or channel strategy.

- Propagation should move from exposure to trust.
- Design content so participants, guests, and media want to retell it.

### Chapter 8: Conversion (转化)

Use when leadership asks "what did this event actually produce?"

- Track not only heat, but also retained relationships and reusable assets.
- Each event should create a bridge to the next conversation, not end at the closing slide.

### Chapter 9: Tools and AI (工具与 AI)

Use when the user needs workflow design, AI prompt chains, knowledge capture, or operating templates.

- Build a mother document before producing outward-facing deck-style plans.
- Use AI to reduce repetitive work, not to outsource judgment.

### Chapter 10: Growth

Use when coaching event operators or market team members.

- Methods matter, but the operator matters more.
- Responsibility, self-drive, taste, calm execution, and "work as craft" are recurring themes.
